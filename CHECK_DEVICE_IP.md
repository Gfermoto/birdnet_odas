# Проверка IP адреса устройства

## Проблема
Устройство недоступно по старому IP адресу 192.168.1.136

## Решение 1: Проверка через роутер

### Шаг 1: Войти в веб-интерфейс роутера
- Обычно: http://192.168.1.1 или http://192.168.0.1
- Логин/пароль: обычно на наклейке роутера

### Шаг 2: Найти список подключенных устройств
- Обычно в разделе: "DHCP Clients", "Connected Devices", "Network", "LAN"
- Ищите устройство с именем "NanoPi-M4B" или MAC адресом

### Шаг 3: Проверить новый IP адрес
- Устройство могло получить новый IP от DHCP
- Попробуйте подключиться по новому IP

## Решение 2: Сканирование сети

Если у вас есть доступ к другой машине в той же сети:

```bash
# Linux/Mac
nmap -sn 192.168.1.0/24

# Windows (если установлен nmap)
nmap -sn 192.168.1.0/24
```

Или через PowerShell:
```powershell
1..254 | ForEach-Object { Test-Connection -ComputerName "192.168.1.$_" -Count 1 -Quiet }
```

## Решение 3: Проверка через другие сервисы

Если были настроены:
- **ZeroTier VPN** - попробуйте подключиться через него
- **Wi-Fi** - если устройство подключено по Wi-Fi, проверьте другой IP
- **Web интерфейс BirdNET-Go** - попробуйте http://192.168.1.x:8080 для разных x

## Решение 4: Подождать

Иногда устройства автоматически перезагружаются или сеть восстанавливается:
- Подождите несколько часов
- Попробуйте подключиться снова
